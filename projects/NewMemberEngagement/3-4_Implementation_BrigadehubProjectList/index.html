Implementation: Project List Improvement


Code for SF new members need a project list that displays key information about the projects in a user-friendly way to help them identify projects of interest at-a-glance.


Key project information to include in the project list. The research group identified factors that influence new members’ decisions on participating in a project by reviewing earlier user interviews with new members. We then conducted card sorting to evaluate the priority and importance of these factors. The key project information new members look for when choosing projects to work on are listed below.

  • Top priority factors:
    • Mission statement
    • Skills used in the project
    • Learning opportunities
    • To-do tasks
  • Lower priority factors (but also important):
    • Past deliverables/progress
    • Expected level of participation
    • Team lead’s name/photo/contact information

Usability improvementThe research group coordinated with BrigadeHub, the development team responsible for the website, to conduct storyboarding and user testing on the current website to identify areas of improvements. The user testing results and suggestions are published and shared with BrigadeHub.


The project needs help with implementing of the UI changes and usability improvements. If you are a skilled front-end developer interested in this project, please contact the Research Group.
